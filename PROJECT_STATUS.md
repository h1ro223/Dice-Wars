**最終更新: 2026-04-16 12:25**

## プロジェクト概要

崩壊：スターレイルのイベントミニゲームを参考に作ったサイコロカードバトルゲーム。
HTML/CSS/JS のみで動作。オンライン対戦はNode.js+WebSocketサーバー（Renderでホスト）。

- **GitHub:** https://github.com/h1ro223/Dice-Wars
- **プレイURL:** https://h1ro223.github.io/Dice-Wars/
- **オンラインサーバーURL:** https://dice-wars-0rcs.onrender.com

### v1.7での主な変更（2026-04-16）

#### 🎮 新ゲームモード3種追加

##### 🎲 カオスモード
- VS CPU・ローカルマルチ両対応
- 毎ターン、両プレイヤーのサイコロ構成が **4D / 6D / 8D からランダム** に変わる
- キャラ固有のATK/DEF/HP/能力はそのまま有効
- `generateChaosDice(count)` 関数でランダムサイコロを生成
- `GS.chaos` フラグ（true/false）で制御
- `setupP1Dice()` / `setupP2Dice()` / `cpuPhase()` で `GS.chaos` 時にランダム構成を使用

##### ⚔️ 3on3 チームバトルモード
- VS CPU・ローカルマルチ両対応
- **キャラ選択画面**: 最大3体をタップで選択（選択順に番号バッジ 1・2・3 表示。同じキャラ再タップで解除）
- **CPU対戦時**: ステップ2に進んだ時点でCPUがランダムに3体自動選択・表示
- **バトル進行**: リレー形式（1体目同士で開戦→どちらかHP0になったら次のキャラに交代）
  - 交代したキャラはHP満タンで登場（勝利側キャラのHPはそのまま引き継ぎ）
  - キャラ交代時は **「○○ DOWN! Next: △△」演出**（2秒）が表示される
  - 3体全員HP0になった側の負け（両者同時→引き分け）
- **チームインジケータ**: バトル中のプレイヤーエリアに残チームを絵文字で表示（現在のキャラ=金枠、脱落=グレーアウト）
- **GS.teamMode** フラグ（true/false）で制御
- GS追加フィールド: `p1Team[]`, `p2Team[]`, `p1TeamIdx`, `p2TeamIdx`
- 主要追加関数: `advanceTeam()`, `showTeamTransition()`, `startNextRound()`, `updateTeamBars()`

##### 🔥 ハードコアCPUモード
- **VS CPU専用**（ローカルマルチやオンラインには影響しない）
- CPU側のHP が **+5** されてバトル開始
- CPU AIの**リロール閾値強化**: 通常 `d.value <= d.type / 2` → ハードコア `d.value <= d.type * 0.6`
- CPU AIの**サイコロ選択強化（ガルム・ジャスパーなど攻撃能力持ちキャラのみ）**:
  - 全組合せ（`getCombinations()`）を列挙して能力ボーナスを計算し最大値を選択
- **GS.hardcore** フラグ（true/false）で制御
- 主要追加関数: `hardcoreSmartSelect()`, `getCombinations()`

#### 🎨 UI/UX改善（v1.7）
- **VS CPUボタン** をクリックすると「⚔️ VS CPU」モード選択モーダルが開くように変更
  - 通常バトル / カオスモード / 3on3チームバトル / ハードコア の4択
- **ローカルマルチ** にもモード選択モーダルを追加
  - 通常バトル / カオスモード / 3on3チームバトル の3択（ハードコアはCPU専用なので除外）
- **バトルHUDにモードバッジ**追加: カオス時「🎲 カオス」（紫）、3on3時「⚔️ 3on3」（オレンジ）、ハードコア時「🔥 ハード」（赤）
- **リザルト画面**にモード名を付記（例: `1P ガルム の勝利！ [カオス]`）

#### ⚙️ 内部改善（v1.7）
- `setModeAndStart(mode, chaos, hardcore, team)` 関数でモードごとのフラグ一括セット
- `updateModeBadge()` でHUDバッジ管理
- `updateTeamCardVisuals(gid, team)` でデッキ選択画面の選択状態+番号バッジ管理
- `checkReady()` にチームモード判定を追加
- `resetBattleState()` にチームバー・モードバッジのリセット処理を追加

### v1.5.1での主な変更
- **音響システムを Web Audio API (AudioContext + AudioBuffer) に完全移行**
  - HTMLAudioElement (`new Audio`) → `AudioContext` + `AudioBufferSourceNode`
  - iOS SafariのSE再生遅延を解消
  - Dynamic Island / メディアコントロールへの波形表示を防止 (`mediaSession.playbackState='none'`)
  - 全音声ファイルを `fetch` + `decodeAudioData` でメモリにプリロード
  - BGM: `GainNode` で音量制御 + `linearRampToValueAtTime` でフェードイン
- **スマホBGM再生修正**: プリロード完了前のタップでも後続操作でBGMが再生されるように修正
- **ホバーSE重複再生修正**: キャラカード内の子要素間移動でCursor.mp3が繰り返し再生されるバグを修正
- **ターン概念変更**: 1ターン = 両者が攻撃・防御を行う（以前は片方の攻防で1ターン）

## オンラインマルチプレイの実装状況

### アーキテクチャ（最新版）
**「ロビーマッチング方式」** に全面刷新済み（ルーム制は廃止）

#### 接続フロー
1. URLをデフォルト値 `wss://dice-wars-0rcs.onrender.com` で接続
2. プレイヤー名（英数字6文字以内）を入力して「登録」
3. ロビー画面でオンライン中のプレイヤー一覧が表示される
4. 対戦したいプレイヤーをタップ→相手も自分を選ぶと**確認ポップアップ**が表示
5. **先に選択した側がホスト**になる（「⭐ あなたはホストです」表示）
6. 両者が「はい」→ デッキ選択画面へ（どちらかが「いいえ」→ ロビーに戻る）
7. デッキ選択（ランダム不可、オンライン時は自分の分のみ）→ バトル開始

#### ホスト基準の画面統一
- P1(下) = ホストのキャラ、P2(上) = ゲストのキャラ
- **両者の画面で同じキャラ配置**を表示
- ホストはP1(下)を操作、ゲストはP2(上)を操作
