## プロジェクト概要

崩壊：スターレイルのイベントミニゲームを参考に作ったサイコロカードバトルゲーム。
HTML/CSS/JS のみで動作。オンライン対戦はNode.js+WebSocketサーバー（Renderでホスト）。

- **GitHub:** https://github.com/h1ro223/Dice-Wars
- **プレイURL:** https://h1ro223.github.io/Dice-Wars/
- **オンラインサーバーURL:** https://dice-wars-0rcs.onrender.com

### v1.5.1での主な変更
- **音響システムを Web Audio API (AudioContext + AudioBuffer) に完全移行**
  - HTMLAudioElement (`new Audio`) → `AudioContext` + `AudioBufferSourceNode`
  - iOS SafariのSE再生遅延を解消
  - Dynamic Island / メディアコントロールへの波形表示を防止 (`mediaSession.playbackState='none'`)
  - 全音声ファイルを `fetch` + `decodeAudioData` でメモリにプリロード
  - BGM: `GainNode` で音量制御 + `linearRampToValueAtTime` でフェードイン
- **スマホBGM再生修正**: プリロード完了前のタップでも後続操作でBGMが再生されるように修正
- **ホバーSE重複再生修正**: キャラカード内の子要素間移動でCursor.mp3が繰り返し再生されるバグを修正
- **ターン概念変更**: 1ターン = 両者が攻撃・防御を行う（以前は片方の攻防で1ターン）
- バックアップファイル追加: `script_backup_v1.5_audio.js`（Web Audio API移行前の完全復元用）

## オンラインマルチプレイの実装状況

### アーキテクチャ（最新版）
**「ロビーマッチング方式」** に全面刷新済み（ルーム制は廃止）

#### 接続フロー
1. URLをデフォルト値 `wss://dice-wars-0rcs.onrender.com` で接続
2. プレイヤー名（英数字6文字以内）を入力して「登録」
3. ロビー画面でオンライン中のプレイヤー一覧が表示される
4. 対戦したいプレイヤーをタップ→相手も自分を選ぶと**確認ポップアップ**が表示
5. **先に選択した側がホスト**になる（「⭐ あなたはホストです」表示）
6. 両者が「はい」→ デッキ選択画面へ（どちらかが「いいえ」→ ロビーに戻る）
7. デッキ選択 → バトル開始（ホストのみ開始ボタンを押せる）

#### コインフリップの同期
- ホストがコイン結果を生成 → `start_battle` メッセージで送信
- ゲスト側は**反転せず**ホストと同じ値を使用
- ホストのボタン：「開始」（ゲスト準備待ちでdisabled）
- ゲストのボタン：「準備OK」→ 押すとホストの「開始」有効化 → 同時開始

#### ホスト基準の画面統一
- P1(下) = ホストのキャラ、P2(上) = ゲストのキャラ
- **両者の画面で同じキャラ配置**を表示
- ホストはP1(下)を操作、ゲストはP2(上)を操作

#### サーバー（server/server.js）の概要
- プレイヤー名をキーに `Map` で管理
- `pendingMatches` マップで対戦確認中のマッチを管理
- 接続時: `set_name` → 名前登録 → `player_list` をロビー全員にブロードキャスト
- 選択時: `select_opponent` → 相互選択で `match_request` を両者に送信
- 確認時: `match_accept` → 両者承認で `match_created` を送信
- 辞退時: `match_decline` → 相手に `match_declined` を通知してロビーに戻す
- ゲーム中: `game_action` をラップして相手に転送するリレー方式
- 切断時: マッチ相手に `opponent_disconnected` を通知

#### クライアント同期の仕組み（script.js）
- `isMyTurn()` でホスト/ゲストのターンを判定
- ダイスロール時: 自分側のみ乱数生成 → `dice_values` で相手に送信
- ダイス選択時: `dice_toggle` を送信して相手画面も更新
- リロール時: `reroll` でリロール後の全ダイス値を送信
- OKボタン時: `confirm` でフェーズ進行を同期（連打防止あり）
- 切断時: もう片方もWebSocketを閉じてタイトルに戻る

## 別PCで作業を再開するには

1. GitHubからクローン or ZIPダウンロード
2. `Dice Wars/server/` フォルダで `npm install`
3. ローカルテスト: `node server.js` → `ws://localhost:8080` で接続
4. VS Codeの「Live Server」拡張でindex.htmlを開く
5. VSCode拡張機能は `installed_extensions.md` を参照

**AIに引き継ぐ場合の伝え方:**
> 「Dice Wars v1.5のオンラインマルチプレイの続きです。PROJECT_STATUS.mdとinstalled_extensions.mdを読んで状況を把握してください。今やりたいことは〇〇です。」

### 音声ファイルについて
音声ファイル（BGM/・SE/フォルダ）は手動でアップロードが必要。
コードはファイルが存在しない場合もエラーにならない（`.catch(()=>{})` 処理済み）。
